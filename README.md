# TCS NQT Advanced Coding Repository

## Overview
This repository serves as a dedicated workspace for mastering **TCS NQT Advanced Coding** challenges. The objective is to bridge the gap between complex, story-based problem statements and foundational Data Structures and Algorithms (DSA).

Every solution contained herein is engineered to meet the strict constraints of the TCS iON evaluation engine, with a primary focus on **optimal time and space complexity**.

## Engineering Methodology
To maintain consistency and technical rigor, each solution adheres to the following criteria:

*   **Complexity Analysis:** Solutions prioritize $O(N)$ or $O(N \log N)$ time complexities to ensure execution well within the specified limits.
*   **Memory Optimization:** Emphasis on $O(1)$ space complexity by leveraging in-place manipulation, two-pointer techniques, and on-the-fly I/O processing.
*   **Edge Case Resilience:** Rigorous handling of boundary conditions (e.g., empty arrays, single-element inputs, and integer overflow) to pass hidden test cases.
*   **Standard I/O Compliance:** Solutions are designed for Standard I/O (cin/cout), bypassing high-level abstractions to ensure zero-overhead execution in the TCS IDE.

## Technical Standards
- **Language:** C++ (Standard 17/20)
- **Programming Paradigm:** Efficient, pattern-based logic (Two-Pointers, Sliding Window, Bit Manipulation, DP, Greedy).
- **Execution Strategy:** Each problem is mapped to its underlying algorithmic pattern to build intuition for unseen "story-based" problems during the actual examination.

## Repository Usage
This workspace is designed to be self-sustaining. As new problems are added:
1.  **Analyze:** Identify the underlying DSA pattern (e.g., DP, Two-Pointers).
2.  **Optimize:** Draft the solution focusing on time and space bounds.
3.  **Validate:** Test against custom edge cases including large inputs and boundary values.
4.  **Commit:** Follow the established commit conventions to maintain a clean project history.
